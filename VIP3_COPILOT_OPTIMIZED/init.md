# VIP3 Workflow Initialization

## Project Goals
- Establish a cohesive development workflow.
- Enhance collaboration among team members.
- Streamline onboarding processes for new contributors.

## Tech Stack
- **Frontend:** React, Redux
- **Backend:** Node.js, Express
- **Database:** MongoDB
- **DevOps:** Docker, GitHub Actions

## Workflow Principles
1. **Continuous Integration:** Ensure all code changes are automatically tested and merged.
2. **Code Reviews:** Implement peer reviews for all pull requests to maintain code quality.
3. **Documentation:** Keep all project documentation up to date and easily accessible.

## Sprint Overview
- Each sprint will last for 2 weeks.
- At the end of each sprint, we will hold a retrospective to discuss what went well and what areas need improvement.

## Guidelines for Users
- Always branch off from the `main` branch for new features.
- Write clear commit messages and pull request descriptions.
- Update documentation as needed when making changes.

## Guidelines for GitHub Copilot
- Generate comments and documentation to explain complex code blocks.
- Provide suggestions for code optimizations and best practices.
- Notify users about the use of Copilot to maintain transparency in code generation.
